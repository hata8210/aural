   * 数据库类型: PostgreSQL (版本 17)。
   * 运行/存储方式: 使用 Docker 容器在本地运行。你可以通过 npx supabase start 启动服务，数据持久化在 Docker 卷中。

  2. 数据库链接信息 (本地开发环境)
  根据 .env.example 和配置文件，标准的数据库连接信息为：

   * 数据库连接串 (Direct Connection): 
      postgresql://postgres:postgres@127.0.0.1:54322/postgres
   * 主机 (Host): 127.0.0.1 (运维手册建议使用 IP 以规避代理干扰)
   * 端口 (Port): 54322
   * 用户名 (User): postgres
   * 密码 (Password): postgres
   * 数据库名 (DB Name): postgres

  3. API 与 管理界面信息
   * Supabase API 地址: http://127.0.0.1:54321 (用于前端 SDK 调用)
   * Supabase Studio (Web 管理后台): http://127.0.0.1:54323 (可以直接在浏览器打开查看和管理数据表)
   * 邮件测试服务器 (Inbucket): http://127.0.0.1:54324



通过 Supabase 的初始化迁移脚本（`supabase/migrations/`）可以非常清晰地了解到数据库的结构。

以下是根据 `001_initial_schema.sql`、`002_activity_segments.sql` 和 `004_interview_prep.sql` 整理的数据表及其作用说明：

## 1. 核心架构表 (Core Infrastructure)

这些表构成了系统的基础权限和组织架构。

| 表名 | 作用说明 | 关键字段 |
| --- | --- | --- |
| **profiles** | **用户档案**：存储用户的基本信息。 | `id` (关联 `auth.users`), `email`, `name`, `role` (USER/ADMIN/ENTERPRISE) |
| **organizations** | **组织/空间**：租户系统的顶层结构。 | `id`, `name`, `slug` (唯一标识), `ownerId` |
| **organization_members** | **组织成员**：管理用户与组织的关系。 | `workspaceId`, `userId`, `role` (OWNER/ADMIN/MEMBER/VIEWER) |
| **projects** | **项目**：组织下的二级分类，用于归类面试。 | `id`, `organizationId`, `name`, `createdBy` |

---

## 2. 面试业务表 (Interview Engine)

面试的核心业务逻辑存储在这些表中。

| 表名 | 作用说明 | 关键字段 |
| --- | --- | --- |
| **interviews** | **面试模板/配置**：定义一场面试的规则。 | `title`, `mode` (CHAT/VOICE/HYBRID), `aiPersona`, `jobDescription`, `resumeText` |
| **questions** | **题目库**：具体的面试题目。 | `interviewId`, `text`, `type` (CODING/WHITEBOARD等), `starterCode` |
| **candidates** | **候选人**：预设或邀请的面试人信息。 | `name`, `email`, `inviteToken` (邀请码), `sessionId` (关联面试会话) |

---

## 3. 会话与记录表 (Sessions & Interaction)

记录面试过程中的动态数据。

| 表名 | 作用说明 | 关键字段 |
| --- | --- | --- |
| **sessions** | **面试会话**：单次面试实例的状态。 | `status` (IN_PROGRESS/COMPLETED), `summary`, `activitySegments` (活跃时长统计) |
| **messages** | **消息记录**：面试中的对话流。 | `sessionId`, `role` (USER/ASSISTANT), `content`, `audioUrl` (录音地址) |

---

## 4. 面试练习模块 (Interview Prep)

用于用户自我练习的扩展表（由 `004_interview_prep.sql` 定义）。

| 表名 | 作用说明 | 关键字段 |
| --- | --- | --- |
| **prep_sessions** | **练习会话**：用户自主发起的练习记录。 | `interviewId`, `userId`, `status`, `durationLimitMinutes` |
| **prep_attempts** | **练习尝试**：用户针对某一题的具体回答。 | `sessionId`, `questionId`, `answerText`, `feedback` (AI评分/建议), `score` |

---

## 5. 系统辅助表 (System Helpers)

| 表名 | 作用说明 |
| --- | --- |
| **api_keys** | 外部集成用的 API 密钥管理。 |
| **audit_logs** | 系统操作审计日志。 |
| **webhooks** | 外部系统回调配置。 |
| **support_tickets** | 用户支持工单。 |

---

### 💡 总结提示：

* **命名规范：** 数据库采用了 `camelCase`（驼峰命名法）以匹配 TypeScript 代码。
* **安全性：** 所有表都启用了 **RLS (Row Level Security)**，这意味着查询时必须考虑当前登录用户的权限（由脚本中的 `POLICY` 定义）。
* **枚举类型：** 脚本定义了许多自定义类型（如 `InterviewMode`, `QuestionType`），在编写 SQL 查询时需要注意这些预定义的取值范围。

> ℹ️ **提示：** 如果您想针对某个特定表编写导出脚本，可以参考 `db_demo/query_to_csv.py` 中的方式，只需修改 `default_query` 即可。
